# ✅ 表格增强：新增"核心观点"列

## 🎯 功能说明

现在报告中的每篇文章都会自动生成**一句话核心观点摘要**！

### 📊 表格结构（4列）

| 列名 | 宽度 | 说明 |
|------|------|------|
| 标题 | 30% | 文章/论文标题（可点击） |
| **核心观点** | 35% | **AI 生成的一句话摘要** |
| 来源 | 20% | 数据源名称 |
| 发布时间 | 15% | 发布日期 |

### 🤖 AI 摘要生成流程

1. **内容获取**
   - 对于 arXiv 论文：使用论文 abstract
   - 对于博客文章：尝试获取文章正文（前3000字符）
   - 备选方案：使用 RSS feed 的 summary

2. **智能摘要**
   - 使用 Claude AI 生成一句话摘要（30-50字）
   - 提取核心观点和创新点
   - 语言简洁专业

3. **错误处理**
   - 如果网站访问受限（403/401），使用 RSS summary
   - 如果内容不足，显示"暂无摘要"
   - 自动重试和降级策略

## ⚙️ 配置说明

**API 调用量**
- 每篇文章 1 次 API 调用（生成摘要）
- 30 篇文章 = 30 次 API 调用
- 建议使用 `max_items_per_category` 控制数量

**当前配置**
```yaml
report:
  max_items_per_category: 10  # 每类10篇 = 共30篇 = 30次API调用
```

## 💡 使用建议

1. **首次运行**：建议设置 `max_items_per_category: 3` 测试
2. **正式使用**：设置为 `10` 获取完整报告
3. **API 成本**：每篇摘要约消耗 200 tokens（输入+输出）

## 📊 运行时间估算

- 数据获取：~10秒
- 文章摘要生成：~30秒（30篇，每篇1秒）
- 类别摘要：~10秒
- 洞察分析：~5秒
- **总计：约1分钟**
